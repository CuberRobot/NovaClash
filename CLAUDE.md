## 项目概述

NovaClash 是一个回合制策略桌游模拟器，用于测试角色平衡性和游戏机制。两名玩家各获得 6 个角色的角色池，每回合选择 3 个角色，应用 4 次增益（ATK+2 或 HP+4），进行三局两胜制的对战。

**当前版本：v1.4**

## 常用命令

```bash
# 运行命令行版本（本地双人同屏）
python main.py

# 运行 Web 服务器（WebSocket 在线对战）
uvicorn webserver:app --host 0.0.0.0 --port 8000

# 创建虚拟环境并安装依赖
python3 -m venv .venv && . .venv/bin/activate && pip install -r requirements.txt
```

## 游戏模式

### 标准模式（Standard）
- 角色池：6 选 3
- 增益次数：4 次
- 战斗顺序：A1-3, B1-3

### 混沌模式（Chaos）
- 角色池：6 选 3（排除自爆步兵）
- 增益次数：4 次
- 特色：每个角色随机分配 0-2 个标签，每个玩家 4 个标签
- 互斥规则：自爆相关组合、狂暴 + 重装盔甲

### 大战场模式（Big Battlefield）
- 角色池：9 选 5
- 增益次数：6 次
- 战斗顺序：A1-5, B1-5

## 架构设计

三层架构，清晰分离：

```
┌─────────────────────────────────────────────────────────────┐
│  webserver.py - FastAPI + WebSocket 服务器                   │
│  房间管理、匹配、回合结算、事件广播                          │
├─────────────────────────────────────────────────────────────┤
│  game_service.py - 业务逻辑层                                │
│  角色池生成、队伍构建、战斗编排                              │
├─────────────────────────────────────────────────────────────┤
│  character.py - 核心战斗引擎                                 │
│  Soldier、Battlefield、标签效果、战斗结算                    │
└─────────────────────────────────────────────────────────────┘
```

**数据流：** WebSocket → 房间管理 → `build_team_from_selection()` → `Battlefield.start_battle()` → 事件列表 → `event_to_text()` → 响应

## 12 种标签效果

| 编号 | 标签 | 效果描述 |
| ---- | ---- | -------- |
| 1 | 自爆 | 攻击时自身必死；目标 max HP≤26 直接击杀；自爆伤害不参与护盾 |
| 2 | 群体治疗 | 己方角色被击杀时消耗次数以 30% 最大血量复活（自爆步兵不可被复活） |
| 3 | 标签剥夺 | 战斗开始时剥夺对方一名有标签角色的全部标签 |
| 4 | 护盾 | 为队友分担 50% 伤害至自身，有次数限制 |
| 5 | 重装盔甲 | 受到≥6 点伤害时按 60% 结算 |
| 6 | 箭矢穿透 | 若攻击目标为脆皮 (max HP≤24)，命中后对另一名敌人追加减半伤害 |
| 7 | 狂暴 | HP≤14 时伤害×1.4 |
| 8 | 中毒 | 叠加型 DOT，每轮次造成伤害，可叠加但有上限 |
| 9 | 群伤 | 对对方所有存活单位造成等量伤害 |
| 10 | 吸血 | 造成伤害的 25% 转为回复，单次上限 5；自爆不触发 |
| 11 | 反弹 | 实际受到伤害（重装减伤后）的 40% 反弹给攻击者，反弹伤害不触发护盾/反弹/重装；自爆攻击不触发反弹 |
| 12 | 斩杀 | 目标当前 HP ≤ max HP 25% 时，伤害×1.5 |

## WebSocket 接口（webserver.py）

### 连接
- `GET /` - 返回前端页面
- `WS /ws` - WebSocket 连接

### 客户端 → 服务器消息

| 类型 | 参数 | 说明 |
| ---- | ---- | ---- |
| `join_random` | name | 随机匹配，返回房间号 |
| `join_room` | name, room | 加入指定房间（4 位代码） |
| `submit_round` | round, selection[], gains[] | 提交回合：选择 3 角色序号 +4 次增益 |
| `leave_room` | - | 离开当前房间 |

### 服务器 → 客户端消息

| 类型 | 说明 |
| ---- | ---- |
| `match_wait` | 已创建随机房间，等待对手 |
| `match_found` | 找到已有随机房间 |
| `joined` | 成功加入房间，返回玩家编号 |
| `matched` | 两人到齐，对局开始 |
| `round_start` | 新回合开始，下发角色池和比分 |
| `submitted` | 本方已提交 |
| `opponent_submitted` | 对手已提交 |
| `round_result` | 回合结算：events 数组、texts 战报、winner 胜者 |
| `match_end` | 对局结束：winner 最终胜者、score 比分 |
| `left_room` | 离开房间（含 opponent_left 表示对手离开） |
| `error` | 错误消息 |

### 战斗事件类型（events 数组）

`FIRST_STRIKE`, `CURSE_REMOVE_TAG`, `ROUND`, `POISON_TICK`, `POISON_APPLY`, `SKIP_DEAD`, `SELF_DESTRUCT`, `SHIELD_TRANSFER`, `ARMOR_REDUCE`, `PIERCE`, `AOE_HIT`, `HIT`, `DIE`, `REVIVE`, `LIFE_STEAL`, `REFLECT`

## 核心流程

**战斗流程：** 先手判定 (先手值之和，小者先手，相同随机) → 轮次顺序 → 中毒结算 → 攻击顺序
- 标准模式/混沌模式：A1,B1,A2,B2,A3,B3
- 大战场模式：A1,B1,A2,B2,A3,B3,A4,B4,A5,B5
→ 目标选择 (基本策略 + 优先标签) → 伤害计算 → 死亡/复活判定

**混沌模式标签分配：**
- 可用角色：19 名（排除自爆步兵 ID=4）
- 可用标签：11 种（排除自爆 TAG=1）
- 每个玩家 4 个标签，每个角色最多 2 个标签
- 互斥规则：(1,2) 自爆 + 复活，(1,10) 自爆 + 吸血，(1,7) 自爆 + 狂暴，(7,5) 狂暴 + 重装

**状态管理：** 所有状态存储在内存中。房间为每位玩家固定角色池 (整场对局不变)，每回合重置提交数据。三局两胜，对局结束或断线时自动清理房间。
