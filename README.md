# NovaClash

**当前版本：v1.2**

这是一个桌游的实验脚本，用于测试角色平衡性和运行的可行性，方便进行模拟和调整。

## v1.2 新特性

- **重构后端架构**：所有玩法与标签判定逻辑集中在 `character.py`，外部输入/输出通过中间层转换，便于后续扩展
- **新增角色**：扩展至 20 名角色，包含更多标签组合
- **游戏平衡性增强** 通过修改特性提升了游戏的平衡性和可玩性

## 快速开始

### 命令行模式

```bash
python main.py
```

### Web 模式
```bash
pip install -r requirements.txt
uvicorn webserver:app --host 0.0.0.0 --port 8000
```
访问 http://localhost:8000

## 核心玩法（v1.2）

游戏总共有两个玩家，三次对局。在游戏开始的时候每个玩家会随机抽取一个角色池，角色池中有 6 个不同的角色。每个对局中两个玩家分别在自己的角色池中依次选取三个角色参与战斗，三个角色的排列顺序决定在战斗中的攻击顺序。选取角色后有四次增益的机会给角色进行 ATK 或者 HP 的增益，可以 ATK+2 或者 HP+4。两个玩家无法在选取和增益阶段看到对方的选择。接下来是战斗环节，角色以 A1，B1，A2，B2，A3，B3 的顺序进行攻击，攻击目标选取目标队伍中 HP 最低的角色。玩家将会得到战斗过程的完整反馈。当某个队伍全部角色死亡，本回合结束。先获取 3 个回合中的 2 个回合胜利的玩家取得最终的胜利。玩家需要在对局中进行博弈，猜对方队伍的角色排布和策略并进行角色安排以求取得最终的游戏胜利。

### 标签种类与描述

| 序号 | 种类                 | 描述                                                         |
| :--- | -------------------- | ------------------------------------------------------------ |

## 项目结构与运行方式

### 项目结构一览

```
NovaClashV1/
├── character.py          # 核心战斗引擎（角色表、Soldier、Battlefield、标签效果）
├── game_service.py       # 业务层（角色池、组队、战斗调用）
├── main.py               # 命令行双人对战入口
├── webserver.py          # Web 服务入口（FastAPI + WebSocket）
├── requirements.txt
├── README.md
├── static/               # 前端静态资源
    ├── index.html
    ├── app.js
    ├── styles.css
    └── rules.html        # 规则说明页

```

### 代码结构

- `character.py`  
  - 核心战斗引擎与角色/标签实现。  
  - 包含：
    - 角色表 `ROLES`（第二版 12 名角色，含 ATK/HP/tag 列表）；  
    - `Soldier`：战场单位；  
    - `Battlefield`：战斗流程（根据两个队伍的 Soldier 列表进行回合制战斗）；  
    - 各种标签效果（自爆 / 复活 / 剥夺 / 护盾 / 重装 / 穿透 / 狂暴 / 中毒 / 群伤等）；  
    - 战斗结果以 **事件标记列表** 输出，方便前后端解析。

- `game_service.py`  
  - 数据处理层（后端 API 逻辑的核心）：  
    - `create_player_pools(pool_size=6)`：为两个玩家各生成一份随机角色池；  
    - `build_team_from_selection(pool, indices, gains)`：根据角色序号和 4 次增益选择，构建一支队伍（并做规则校验，如自爆步兵必须在队伍首位、必须恰好 3 人+4 次增益）；  
    - `run_battle_from_teams(team1, team2)`：将两队简单 dict 转成 Soldier，调用 `Battlefield` 跑一局，返回事件列表与胜者。

- `main.py`  
  - 纯命令行测试入口：  
    - 从 `game_service` 获取角色池；  
    - 通过 `input` 让两位玩家依次选择 3 名角色与 4 次增益；  
    - 调用 `run_battle_from_teams` 获取事件标记，再用 `event_to_text` 转成中文战报打印出来；  
    - 实现 3 局 2 胜制的完整对战模拟。

- `webserver.py`  
  - 基于 FastAPI + WebSocket 的 Web 后端：  
    - HTTP：`GET /` 返回前端页面 `static/index.html`；  
    - WebSocket：`/ws`，支持：
      - 随机匹配：`join_random`（现在只负责「查找/创建随机房间并返回房间号」）；  
      - 房间匹配：`join_room`（加入指定房间号，满两人后广播 `matched` 并进入对战）；  
      - 回合提交：`submit_round`（两边都提交完后调用 `run_battle_from_teams`，并把事件/战报广播给双方）。  
    - 已添加基本的状态保护：单个连接不能重复匹配/重复进入房间。

- 前端静态资源 `static/`  
  - `index.html`：单页前端 UI，包含：
    - 匹配/房间管理区（随机匹配、创建房间、加入房间）；  
    - 角色池展示区（卡片式，显示 ATK/HP/标签）；  
    - 出击队伍与 4 次增益配置区；  
    - 战报文本 + 原始事件 JSON 的展示区。  
  - `styles.css`：暗色、玻璃拟态风格的布局与样式。  
  - `app.js`：通过 WebSocket 与 `webserver.py` 通信，完成：
    - 随机匹配 → 获取房间号 → 加入房间；  
    - 房间号加入匹配；  
    - 本回合出阵与增益的提交；  
    - 实时战报渲染与事件标记输出。

### 本地运行方式

1. 创建虚拟环境并安装依赖：

   ```bash
   python3 -m venv .venv
   . .venv/bin/activate
   pip install -r requirements.txt
   ```

2. 运行命令行版本（本地双人同屏测试）：

   ```bash
   python3 main.py
   ```

3. 运行 Web 版本（浏览器联机测试）：

   ```bash
   . .venv/bin/activate
   uvicorn webserver:app --host 127.0.0.1 --port 8000
   ```

   然后在浏览器中打开 `http://127.0.0.1:8000`：  
   - 可以选择 **随机匹配**（由后端分配房间号）；  
   - 或使用 **房间号** 创建/加入房间；  
   - 完成选角与增益后，提交本回合并查看完整战报。
