## 新增内容

### 混沌模式（Chaos Mode）

全新的随机标签玩法，每局游戏角色标签随机分配！

**核心规则**：
- 角色池：6 选 3（排除自爆步兵）
- 增益次数：4 次
- 标签分配：每个玩家 4 个标签，随机分配到 6 个角色上
- 每角色最多 2 个标签
- 可用标签：11 种（排除自爆 TAG=1）

**互斥规则**：
- 自爆 + 群体复活
- 自爆 + 吸血
- 自爆 + 狂暴
- 狂暴 + 重装盔甲

**技术实现**：
- 新增 `CHAOS_AVAILABLE_ROLES` 常量（19 名角色，排除自爆步兵）
- 新增 `generate_chaos_pool()` 函数
- 标签分配算法确保总标签数 = 4

### 大战场模式（Big Battlefield）

5v5 大规模战斗！

**核心规则**：
- 角色池：9 选 5
- 增益次数：6 次
- 战斗顺序：A1→B1→A2→B2→A3→B3→A4→B4→A5→B5
- 胜利条件：三局两胜

**技术实现**：
- `Battlefield` 类支持动态 `team_size` 参数
- `build_team_from_selection()` 新增 `team_size` 参数
- 大战场模式玩家只能与大战场模式玩家匹配

### 房间模式隔离

不同游戏模式之间完全隔离，确保对局公平性！

**实现细节**：
- `Room` 类新增 `mode` 字段
- `GameMode` 枚举：`standard`, `chaos`, `big_battlefield`
- 随机匹配时只匹配相同模式的房间
- 加入房间时校验模式一致性

## 技术细节

### 混沌模式标签分配算法

```
输入：pool_size=6, total_tags=4, max_per_role=2
输出：6 个角色，总标签数=4，每个角色 0-2 个标签

分配流程:
1. 从 CHAOS_AVAILABLE_ROLES 中选择 12 个角色（双方不重复）
2. 阶段 1：优先给每个角色分配 1 个标签（如可能）
3. 阶段 2：随机填充剩余标签，遵守互斥规则
4. 总标签数 = 4 的分配示例：
   - 方案 A：4 个角色各 1 标签，2 个无标签
   - 方案 B：2 个角色各 2 标签，4 个无标签
```
